# HR 경영진 Agent — 개발 기획 · 설계서 (v1)

작성일: 2026-06-19 · 대상: 개발/HR 협업 · 단계: Phase 1 (MVP)

---

## 1. 개요

경영진의 회의 중 빠르고 정확한 데이터 기반 의사결정을 지원하는 **대시보드형 HR AI 비서**.

- **목적**: 임원 회의 시 기다림 없이 즉각적인 인사 데이터 확인 및 본부장 지시사항 진행 현황 팔로업
- **기대효과**: 데이터 조회 시간 단축, 감이 아닌 데이터 기반 의사결정, 리더십 지시사항 누락 방지
- **핵심 성공기준**: 권한에 맞는 데이터를 **단 한 번의 오류도 없이** 정확히 제공 (데이터 정합성 + 보안 통제 무결성)

---

## 2. 사용자 & 권한 매트릭스

핵심 원칙: 본인이 소속/담당하는 조직의 데이터만 열람 가능하도록 철저히 분리.

| 역할 | 열람 가능 | 열람 불가 |
| --- | --- | --- |
| 본부장(임원) | 본인 관할 본부 및 산하 조직 전체 HR 데이터·지시사항 | 타 본부의 개인화된 인사/급여 데이터 |
| HR 시스템 관리자 | 데이터 업로드·시스템/권한 관리·AI 프롬프트 관리 | 별도 권한 미부여 시 챗봇에서 개별 본부 데이터 조회 불가 |

전사 통계(연령대 구성비 등)는 **익명·집계(%) 형태로만** 모든 임원에게 제공.

### 데모 조직 구조

```
영업본부 (SALES, 본부장 김영업)        경영지원본부 (MGMT, 본부장 박지원)
 ├─ 영업1팀 (SALES_1)                  ├─ 인사팀 (HR)
 ├─ 영업2팀 (SALES_2)                  ├─ 재무팀 (FIN)
 └─ 영업기획팀 (SALES_PLAN)            └─ 총무팀 (GA)
```

## 3. 대표 유스케이스 10선 → 구현 매핑

| # | 임원 질문 | 사용 데이터셋 | 챗봇 intent | 대시보드 카드 |
| --- | --- | --- | --- | --- |
| 1 | 퇴사위험 높은 핵심 인재? | employees(평가·근태·위험) | retention | 퇴사위험 핵심 인재 |
| 2 | 영업팀 인력 재배치 진행? | directives | directives(재배치 필터) | 임원 지시사항 트래킹 |
| 3 | 분기 인건비 예산 대비 지출? | budget | budget | 인건비 예산 대비 지출 |
| 4 | 핵심 포지션 채용 진행률? | recruiting | recruiting | 채용 진행 현황 |
| 5 | 초과근무 가장 많은 팀? | employees(overtime) | overtime | 팀별 초과근무 |
| 6 | 하위등급 직원 개선 현황? | pip | pip | 개선 프로그램(PIP) |
| 7 | 영업팀 승진 대상자? | promotion | promotion | 승진 대상자 |
| 8 | 팀장 다면평가 추이? | multi_eval | multieval | 다면평가 추이 |
| 9 | 본부 연령대 구성비? | employees + company_age_pct | age | 연령대 구성비(본부 vs 전사) |
| 10 | MZ세대 선호 복리후생? | (사내 데이터 없음) | external fallback | — |

10번은 사내 데이터 부재 → "현재 사내 데이터에는 없습니다. 다만 일반적인 인사 트렌드에 비추어 볼 때…" 형식으로 **외부 지식임을 명시**하고 조언.

## 4. 시스템 아키텍처 (MVP)

```
┌──────────────────────────────────────────────────────────┐
│  브라우저 (Next.js App Router · 클라이언트 렌더)            │
│                                                            │
│  Login ──► 세션(역할/부서코드)                              │
│    │                                                       │
│    ▼                                                       │
│  Console ──┬── Dashboard (Recharts 시각화)                  │
│            └── ChatPanel ──► agent.ts (룰 기반 RAG)         │
│                  │                                          │
│            access.ts: getScopedData(user)                  │
│            = Row-level Security 필터 (division 기준)        │
│                  │                                          │
│            data.ts ◄── hr_data.json (업로드 데이터)         │
└──────────────────────────────────────────────────────────┘
```

- **권한 게이트**: 모든 데이터 접근은 `getScopedData(user)`를 단일 관문으로 통과. 컴포넌트는 필터링된 스코프만 받으므로 화면 어디서도 권한 외 데이터가 존재하지 않음.
- **그라운딩**: `agent.ts`는 스코프 데이터의 실제 값만으로 답변. 임의 수치 생성 없음.
- **출처 표기**: 답변마다 `internal`(사내) / `external`(외부 일반지식) / `blocked`(권한제한) / `none`(매칭없음) 라벨.

### 향후(서버화) 아키텍처

운영 단계에서는 권한검증을 **서버 측**으로 이전하고, RAG를 LLM + 벡터검색으로 교체. 부서코드 필터를 검색 단계에서 강제(서버 권한토큰 기반)하여 클라이언트 우회 차단.

## 5. 데이터 스키마 (표준 업로드 템플릿)

엑셀 6개 시트 = 6개 데이터셋. 컬럼명이 적재 키. `division`은 모든 시트의 권한 필터 기준.

| 시트 | 주요 컬럼 |
| --- | --- |
| 1.인원마스터 | id, name, division, team, position, age, join_date, tenure_years, eval_grade, retention_risk, monthly_salary_manwon, overtime_hours_3m_avg, is_leader |
| 2.임원지시사항 | id, division, title, owner, progress, status, issue, due_date, requested_by, requested_date |
| 3.인건비예산 | division, team, quarter, budget_manwon, spent_manwon, headcount |
| 4.채용현황 | division, position, stage, applicants, target, progress |
| 5.다면평가 | division, team, name, title, 2025H2, 2026H1, delta |
| 6.개선프로그램PIP | division, team, name, position, grade, program, status |

- 수집 방식: API 연동 배제, HR 담당자가 가공된 엑셀/CSV **수동 업로드**(도입 비용↓, 통제력↑)
- 갱신 주기: 주 1회 또는 정기 회의 전 일괄 업로드
- 민감도: 최상(인사평가·급여·지시사항). 급여 등은 정책에 따라 마스킹 후 업로드.

## 6. 화면 구성 (UX)

```
┌─ 상단바: 브랜드 · 로그인 사용자 · [열람범위 칩] · 로그아웃 ─────────┐
├───────────────────────────────┬──────────────────────────────┤
│  [대시보드] (좌·중앙)          │  [HR Agent 채팅] (우측 고정)  │
│  KPI 4종                       │  · 추천 질문 10선            │
│  퇴사위험 / 초과근무 / 인건비   │  · 출처 라벨 + 표 응답        │
│  지시사항 / 채용 / 연령구성     │  · 질문 시 해당 카드 하이라이트│
│  승진 / PIP / 다면평가          │                              │
└───────────────────────────────┴──────────────────────────────┘
```

- 챗봇 질문 → 관련 대시보드 카드로 자동 스크롤 + 강조(연계 UX).
- 1024px 이하에서 채팅이 하단으로 재배치(태블릿/모바일 대응 기초).
- 관리자 로그인 시 대시보드 대신 **데이터 업로드 + 감사 로그** 패널 표시.

## 7. 보안 · 개인정보 · 컴플라이언스

- **데이터 그라운딩**: 업로드된 실제 숫자만으로 응답. 수치 임의 추산 금지.
- **접근통제**: 사용자 부서코드 식별 → RAG/조회 시 타 부서 데이터 행 단위 필터(Row-level Security).
- **공정성/편향 방지**: AI가 직원의 능력·성과를 임의 평가·추측·프로파일링하는 문구 생성 금지. 사실 데이터만 전달.
- **감사 로그**: 임원 질의 내역 기록(관리자 확인용, 보안 익명화 처리).

---

## 8. 답변 규칙

- **톤**: 임원 보고체(정중·간결·개조식).
- **출처 표기**: 사내 데이터에 없으면 "현재 사내 데이터에는 해당 내용이 없습니다. 다만 일반적인 인사 트렌드에 비추어 볼 때…" 형식으로 외부 지식임을 명시.
- **창조 금지**: 없는 숫자를 임의 추산해 보고하지 않음.

---

## 9. 기능 우선순위

- **P0(본 MVP)**: 권한 분리·인증, 엑셀/CSV 수동 업로드 창구, 대시보드+챗봇 통합 화면, 업로드 데이터 기반 RAG 답변.
- **P1**: 대시보드 지표 커스텀, 정기 요약 리포트 이메일 발송.
- **P2**: 임원별 관심사/자주 묻는 질문 기억(초개인화 학습).

---

## 10. 로드맵

- **Phase 1 (MVP, 본 산출물)**: 수동 업로드 + 권한 분리 + 대시보드·챗봇 통합 화면 배포.
- **Phase 2**: 사내 메신저(슬랙/팀즈) 연동 지시사항 리마인드, 개인화 학습, 서버 측 권한검증 + LLM RAG.
- **Phase 3**: 핵심 시스템(HRIS) 읽기 전용 API 부분 연동 검토.

### 스코프 아웃 (v1 제외)

실시간 HRIS API 연동, 임원별 성향 맞춤형 초개인화 학습.

## 11. HR 협의 필요 사항

- 표준 엑셀/CSV 템플릿(컬럼명·양식) 최종 확정 — 본 설계서 §5 초안 검토
